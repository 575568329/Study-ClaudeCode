# ClaudeCode学习项目设计文档

**设计日期**: 2026-03-26
**设计师**: Claude (AI导师)
**项目类型**: 学习管理系统 (LMS)
**核心理念**: AI驱动 + Obsidian知识管理 + 苏格拉底教学法

---

## 1. 项目概述

### 1.1 项目目标

创建一个系统化的学习项目，用于全面掌握ClaudeCode的能力，包括：
- 基础使用和常用命令
- 高级功能和自动化
- 工具开发和扩展
- 项目实战和最佳实践

### 1.2 核心价值主张

**AI驱动的交互式学习**：
- Claude Code担任私人技术导师
- 苏格拉底教学法引导思考
- 个性化学习路径

**Obsidian知识管理**：
- 双向链接构建知识网络
- 图谱视图可视化学习路径
- 标签系统支持多维度检索

**全面进度跟踪**：
- 每日会话详细记录
- 技能掌握度量化
- 知识盲区识别和优先级排序

### 1.3 设计原则

1. **简单优先**: 复用CFP-Study的成功模式
2. **渐进增强**: 基础功能优先，高级功能逐步添加
3. **知识关联**: 利用Obsidian的双向链接构建知识网络
4. **数据驱动**: 基于学习数据动态调整学习路径

## 4. CLAUDE.md设计

### 4.1 角色定义

**角色名称**: ClaudeCode技术导师

**核心理念**:
- 友好的学习伙伴
- 苏格拉底教学法
- 主动验证理解
- 适应性教学

### 4.2 教学流程

```
用户提问
   ↓
初始探索："你已经了解多少？"
   ↓
[用户回应现有知识]
   ↓
构建解释：基于已知 → 未知
   ↓
简洁说明：~200词 + 代码示例
   ↓
理解检查："你能解释一下..."
   ↓
[用户回应理解程度]
   ↓
适应性跟进：
  - 理解 → 深入/相关概念
  - 部分理解 → 换角度解释
  - 不理解 → 更简单的解释
```

### 4.3 关键行为准则

**✅ 应该做的**：
- 使用对话式语言
- 提供实际代码示例
- 鼓励提问和探索
- 庆祝理解进展
- 提供提示而非直接答案
- 连接到真实场景

**❌ 不应该做的**：
- 一次性倾倒大量信息
- 不检查理解就继续
- 让用户对不懂感到羞愧
- 使用未解释的技术术语
- 直接给出答案而不解释原理

### 4.4 会话记录协议

每次学习会话后，Claude必须完成**两步**：

**Step 1: 更新Obsidian会话笔记**
- 文件：`10-Daily-Sessions/YYYY-MM-DD.md`
- 内容：
  - 会话概览（日期、时长、主题）
  - 每个问题的详细记录
  - 用户初始理解
  - Claude的解释
  - 理解检查结果
  - 掌握程度评估
  - 新掌握的技能列表
  - 发现的知识盲区
  - 下一步计划

**Step 2: 更新进度跟踪**
- 文件：`20-Progress-Tracking/Skills-Mastered.md`
- 操作：
  - 添加新掌握的技能（带日期和置信度）
  - 更新或解决知识盲区
  - 调整学习优先级
  - 更新总体进度百分比
  - 添加双向链接到会话笔记

**Step 3: 创建或更新知识库笔记**
- 文件：`30-Knowledge-Base/[分类]/[主题].md`
- 操作：
  - 创建新概念笔记
  - 添加代码示例
  - 链接到相关会话
  - 添加标签和难度级别

## 6. Obsidian特性应用

### 6.1 双向链接系统

**链接类型**：
1. **会话 → 知识点**：会话中链接学到的概念
2. **知识点 → 会话**：反向链接显示所有相关学习记录
3. **技能 → 会话**：技能掌握跟踪链接到证明会话
4. **知识点 → 知识点**：概念之间的关联

**示例**：
```markdown
在会话笔记中：
今天学习了 [[Read工具]] 和 [[Edit工具]]

在Read工具笔记中：
## 学习历史
- [[2026-03-26]] - 初次学习，掌握了基本用法
- [[2026-03-28]] - 深入学习并行读取
```

### 6.2 标签系统

**分层标签结构**：
```
#技能
  #技能/已掌握
  #技能/学习中
  #技能/待学习

#难度
  #难度/基础 ⭐
  #难度/中级 ⭐⭐
  #难度/高级 ⭐⭐⭐
  #难度/专家 ⭐⭐⭐⭐

#领域
  #领域/CLI
  #领域/Skills
  #领域/Hooks
  #领域/Agents
  #领域/MCP
  #领域/Projects

#状态
  #状态/完成
  #状态/进行中
  #状态/阻塞
```

**使用示例**：
```markdown
---
tags: [skill, 技能/已掌握, 难度/基础, 领域/CLI]
---
```

### 6.3 图谱视图应用

**图谱用途**：
1. **学习路径可视化**：查看已掌握技能的网络
2. **知识关联发现**：发现概念间的隐含联系
3. **复习规划**：按图谱区域系统性复习
4. **盲区识别**：图谱中稀疏的区域是知识盲区

**MOC（Map of Content）笔记**：

```bash
Related: [[2026-03-26-session]]"
```
